# Definition of Done

A Strata feature is not complete until:

- it works as intended
- it has appropriate tests
- it has documentation
- it handles errors gracefully
- it follows project conventions
- it has been reviewed
- README or docs are updated if behavior changed
- changelog is updated if needed
- UI changes include screenshots or visual notes
- security and privacy implications are considered

## Strata-Specific Checks

Telemetry-related work must also answer:

- What data is captured?
- Why is it needed?
- Is sensitive data redacted by default?
- Can teams disable or configure the behavior?
- What is the expected storage and performance impact?
- How does the feature fail safely?
